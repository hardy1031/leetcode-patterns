## Theme: The char lives in the dict key, not in the node

Triggered by 208. Implement Trie. The user initially expected each TrieNode to store the character it represents, plus a children dict. The realization that took time: the node itself doesn't need to know what char it "is" — the parent's dict key already encodes that information.

### Key insight
In a Trie, `node.children['d']` means "the 'd' child exists" — the char is the key, so the node only needs `children` (what comes next) and `is_end` (does a word terminate here).
