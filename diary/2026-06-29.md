## Theme: Why dummy node matters

When implementing 92 (Reverse Linked List II) without a dummy node, the edge case where `left=1` caused `left_node` to be `None`, leading to a crash. By placing a dummy node at the front, `left_node` always exists even when `left=1`, and the reconnection logic can be unified.

### Key insight
The dummy node exists to eliminate edge cases where the head node requires special handling.

## Theme: Set approach vs Fast/Slow approach (141)

Initially solved 141 (Linked List Cycle) using a set — add visited nodes to the set and return True if a node is visited again. This is O(n) space. Using Fast/Slow pointers achieves the same O(n) time with O(1) space. Since fast closes in on slow by one step per iteration, it is guaranteed to catch up if a cycle exists.

### Key insight
The set approach is intuitive but O(n) space. Fast/Slow is O(1) space and preferred in interviews. When multiple approaches exist, being able to explain the space trade-off is a strength.

## Theme: Combining multiple linked list patterns (143)

143 (Reorder List) requires combining three patterns: fast/slow pointers, in-place reversal, and alternating merge. Solved it by watching NeetCode. Fast/slow pointers find the middle of a linked list in O(n) without an index. The merge step requires two temp pointers (tmp1, tmp2) — next nodes must be saved before rewiring pointers, otherwise they are lost.

### Key insight
For combination problems, understanding each individual building block (fast/slow, reverse, merge) is enough — the way to combine them can be derived by drawing a diagram.

- 143. Reorder List
