## Theme: Two Pointers vs Fast/Slow Pointers

Solved 61 (Rotate List) using the gap technique with fast and slow pointers: move fast k steps ahead, then advance both until fast reaches the tail. Both pointers move at the same speed — this is two pointers, not fast/slow. Fast/slow is specifically when fast moves 2x the speed of slow (cycle detection, finding middle). The gap technique (19, 61) keeps a fixed distance between two same-speed pointers.

### Key insight
Same-speed pointers with a fixed gap = two pointers. Different-speed (2x) = fast/slow pointers. The distinction matters for pattern recognition.

## Theme: Dummy Node — When to use, when not to

Solved 19 (Remove Nth Node From End of List) using the gap technique with a dummy node. The dummy node is useful here because when n == sz (remove the head), slow needs to point to a node before head — the dummy provides that anchor. Contrast with 61 (Rotate List) where the dummy node adds no value since the new head position changes dynamically based on k.

### Key insight
Use dummy node when the head itself might be modified or deleted. Skip it when the new head is computed dynamically and the old head anchor is useless.
