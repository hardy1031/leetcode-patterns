## Theme: Top-Down DFS — Helper Function with Extra Arguments

Today's two problems both share the same structural reason for needing a helper function: the public LeetCode method signature takes only `root`, but the recursive logic requires extra state that must be passed down from parent to child.

### Key insight
When the decision at each node depends on information inherited from its ancestors (not computed from its children), you need top-down DFS — and if the public method signature doesn't accept those extra parameters, you wrap the logic in a helper function that does.

### Problems covered
- 1448. Count Good Nodes in Binary Tree — passes `max_so_far` down; each node checks if its value is `>= max_so_far`
- 98. Validate Binary Search Tree — passes `(left_bound, right_bound)` down; the valid range narrows at each level
