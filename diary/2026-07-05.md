## Theme: `if` vs `elif` in while loops — a subtle control flow bug

In LCA of BST, used three `if` statements instead of `if / elif / else`. After `current = current.right`, Python continued evaluating the remaining `if` conditions on the already-updated `current` in the same iteration, causing wrong behavior.

### Key insight
Inside a while loop, if the branches are mutually exclusive and you move a pointer in one branch, use `elif` — not `if`. Using `if` means every condition is evaluated independently, even after a pointer has been moved.

```python
# WRONG — all three ifs run independently in the same iteration
if current.val < small.val:
    current = current.right
if current.val > big.val:       # current already changed!
    current = current.left
if ...:
    return current

# CORRECT
if current.val < small.val:
    current = current.right
elif current.val > big.val:
    current = current.left
else:
    return current
```

## Theme: BST property enables O(h) LCA without visiting all nodes

In a general binary tree, LCA requires O(n) DFS. In a BST, the ordering property means you can decide at each node which subtree must contain the LCA — no backtracking needed.

### Key insight
Label the two targets as `small` and `big`. The LCA is the first node where `small.val <= current.val <= big.val`. This works because BST ordering guarantees that once a node falls between p and q, both p and q are in different subtrees (or one equals the node).
