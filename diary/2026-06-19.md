## Theme: Sort + Duplicate Skip

3Sumを解く中で、「重複したtripletを排除するにはどうするか」という問題にぶつかった。`in result` チェックはO(n)でTLEになると気づき、sortしてpointerをスキップする方法を発見。3箇所（i, left, right）全部にskipが必要という細かい実装ポイントまで掘り下げた。

### Key insight
sortすると同じ値が隣り合うので、`nums[i] == nums[i-1]` のときcontinueするだけで重複をO(1)で排除できる。

## Theme: HashMap vs Set vs Sort — O(n) にできる条件

2Sum（exact match）はHashMapでO(n)、3Sum ClosestはsortなしでO(n)にできない理由を比較した。HashMapは「この値が存在するか」には答えられるが「一番近い値はどれか」には答えられない。3 pointersのアルゴリズム（O(n²)）ではsortのO(n log n)は誤差になる。

### Key insight
「exact matchか？」が分岐点。exact matchならHashMap O(n)、closestならsort必須でO(n log n)が最速。sortのコストが無視できるかどうかは後続アルゴリズムの計算量で決まる。

## Theme: 設計先行 — Pattern → Algorithm → Data Structure を決めてから実装

3SumでHashMapを使ったが、出力が「tripletのリスト（値のみ）」であることを先に確認していればSetで十分だとわかった。コードを書く前にLayer 3→2→1を決める習慣がないと、実装で詰まる。ソフトウェア設計と同じ原則。

### Key insight
「valueを後で使うか？」→ No ならSetで十分。HashMapを選ぶ前に出力の要件を確認する。
