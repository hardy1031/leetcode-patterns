## Theme: DFS for wildcard matching in Trie

Triggered by 211. When search contains `.`, you cannot follow a single deterministic path — you must branch into every child and recurse. The iterative loop breaks down here because you need to "undo" a failed branch and try the next one. DFS naturally handles this: try each child, return True on the first success, return False if all fail.

### Key insight
Convert the iterative search loop to a recursive DFS. Normal chars advance deterministically; `.` fans out to all children with `if dfs(i+1, child): return True`.
