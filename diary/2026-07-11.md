## Theme: When heap beats sorting — streaming and small k

Triggered by solving 1046 and 973 back-to-back, then asking "why use a heap at all?"

### Key insight
Heap wins when **k << n** (O(n log k) vs O(n log n)) or when **data arrives as a stream** — sorting requires all data upfront and re-sorts from scratch, while a heap inserts each new element in O(log k).

## Theme: Heap + deque for cooldown scheduling

Triggered by 621 Task Scheduler — recognized greedy → maxHeap, but couldn't independently see that cooldown tracking needs a deque.

### Key insight
When tasks have a **cooldown window**, pair maxHeap (greedy: always pick most frequent available) with a deque (FIFO cooldown queue storing `(remaining_count, available_at_time)`). The `time` variable acts as the scheduling clock — check `queue[0][1] == time` to re-admit cooled tasks each cycle.
