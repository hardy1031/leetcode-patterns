## Theme: Backtracking = include/exclude binary tree

First backtracking problem. The key realization: at each index, you make exactly two choices — include the element or exclude it — and recurse both. Because i always increases, [1,2] and [2,1] can never both appear. Every leaf of the recursion tree (when i >= len) is a valid subset.

### Key insight
Call backtrack(i+1) twice: once after appending (include path), once after popping (exclude path). The two calls together guarantee every combination is visited exactly once.

## Theme: range(i, len(nums)) prevents duplicate combinations

Triggered by 39. Combination Sum. When the for loop uses `range(len(nums))`, backtracking can return to earlier indices, generating [2,5] and [5,2] as separate results. Using `range(i, len(nums))` enforces a forward-only constraint — once you've moved past index i, you never consider elements before it again, so each combination appears in exactly one order.

### Key insight
`range(i, len(nums))` = no going back to lower indices = no reordered duplicates. Pass the current index `i` (not `i+1`) when the same element can be reused.
