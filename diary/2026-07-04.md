## Theme: "return value ≠ answer" pattern (previously called path_problems)

Clarified the core pattern behind diameter and balance problems. The label "path_problems" is imprecise — balance is not a path problem. The real distinguishing feature is that the recursive return value and the final answer are different things.

### Key insight
The split happens based on whether the answer requires both left and right subtree information simultaneously:
- If the answer can be built bottom-up from a single value → return it directly (basic DFS)
- If the answer requires comparing or combining left AND right at the same node → record it via `nonlocal`, return height as a helper

| | basic | return value ≠ answer |
|---|---|---|
| recursive return | = answer | helper metric (height) |
| answer update | natural propagation | nonlocal side effect |
| examples | max depth, invert | diameter, balance |
